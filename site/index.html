How it works

Paragraph-granular encryption

mdenc splits Markdown on double newlines and encrypts each paragraph independently. When you change one paragraph, only that chunk's ciphertext changes — the rest stays identical.

Deterministic ciphertext

Nonces are derived from HMAC-SHA256 of the plaintext, not random. Same password + same text = same output, every time. This makes encrypted files diff-friendly in git.

File format

mdenc:v1 salt=... file_id=... scrypt=...
hdrauth=<HMAC of header>
<base64 chunk: nonce || ciphertext || tag>
<base64 chunk: nonce || ciphertext || tag>
seal=<HMAC of all lines above>

Git integration

mdenc uses git's smudge/clean filter to transparently encrypt on commit and decrypt on checkout. Your working directory stays plaintext; git stores ciphertext.
